# 钻石商店总体方案

## 一、核心改动：水晶 → 钻石

### 命名规则
| 旧名 | 新名 |
|---|---|
| 水晶 | 钻石 |
| 五色水晶 | 五色钻石 |
| 水晶塔 | 钻石塔 |
| `useCrystal` (hook 名) | **保留不改**，代号沿用 |
| API `/api/crystal/*` | **保留**，避免破坏旧客户端 |
| 数据库字段 `crystals_blue` 等 | **保留** |

### 五色钻石新含义
| 颜色 | Emoji | 来源 | 含义 |
|---|---|---|---|
| 蓝 blue | 💎🔵 | 完成单元（系统赠送）| 学习成就钻 |
| 绿 green | 💎🟢 | 零错误（系统赠送）| 准确成就钻 |
| 红 red | 💎🔴 | 错题坚持（系统赠送）| 毅力成就钻 |
| 紫 purple | 💎🟣 | 连击（系统赠送）| 连胜成就钻 |
| **金 gold** | 💎🟡 | **充值获得**（真金白银） | **充值钻**（核心货币）|

> 关键设计：**金钻石就是付费货币**，其他四色是免费成就钻。商品可分别接受不同色支付。

## 二、充值档位（最终定价）

| 价格 | 金钻数量 | 单价 | 性价比 |
|---|---|---|---|
| ¥1 | 50 颗 | 2 分/颗 | 基础 |
| ¥3 | 160 颗 | 1.875 分/颗 | +6.7% bonus |
| ¥10 | 800 颗 | 1.25 分/颗 | +60% bonus（最优） |
| ¥30 | 3000 颗 | 1 分/颗 | +100% bonus（大额）|

> 算法说明：¥10 档故意做得最划算，引导用户单次充 ¥10；¥30 大额包仍有奖励但收益递减，符合"中档诱导"心理。

## 七、防作弊原则
1. 钻石余额**只信服务器**，前端只展示
2. 商品价格**只信服务器**，前端不许传 price
3. 购买请求带 `client_idempotent_id`，服务端去重
4. 装备宠物前**校验是否在库存里**
5. 充值回调**校验金额 + 订单 + 签名**才发钻

## 八、关键技术细节

### 8.1 现有迅虎支付的复用
- 现有 `/api/payment/create` 用于会员套餐
- 新增 `/api/shop/recharge/create`，逻辑几乎一样，差别：
  - `product_type='diamond_pack'`
  - 回调成功后调 `crystal/earn` 加 gold 钻数
  - 不修改 `is_member`

### 8.2 道具生效机制
- 道具不是"扣余额"型，是"消耗库存"型
- `user_inventory(user_id, item_id, count)`
- 使用时 `POST /api/shop/inventory/use { item_id }` → count-- → 触发对应业务逻辑（如冻结卡 → 调打卡 API 补一天）

### 8.3 装备宠物
- `user_equipped(user_id, slot, item_id)`，slot ∈ {avatar, panda_skin, theme, flame_color}
- 前端读 `equipped.avatar` 渲染顶部 Logo
